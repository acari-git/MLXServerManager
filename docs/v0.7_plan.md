# v0.7 Plan

## Focus

v0.7 focuses on safer switching between multiple saved model profiles.

v0.6 added profile creation and deletion. v0.7 should make it clear which profile is selected in the UI, which profile is actually running, and when a Restart is required before the selected profile takes effect.

Direct Mode remains unchanged:

```text
OpenAI-compatible client -> mlx_lm.server
```

The app continues to manage `mlx_lm.server` directly. It must not proxy inference traffic or add a chat surface.

## Goals

- Make multiple model profiles easier to select.
- Show the selected model profile clearly in the model list.
- Track and display the running model profile when a managed server is active.
- Distinguish selected profile settings from the profile used by the active managed process.
- Apply model selection immediately while the managed server is stopped.
- Allow model selection while the managed server is running, but do not switch the active server immediately.
- Show a `Restart required` state when the selected runtime profile differs from the running profile.
- Make Restart stop the current managed process, wait for port release, then start the selected profile.
- Keep Start using the currently selected profile.
- Keep Stop limited to the current managed process.
- Keep Connection Settings, Copy Config, and copied curl commands following the selected profile.
- Log model switching state changes in a way that explains whether Restart is required.

## Selected Model vs Running Model

v0.7 should make the following concepts explicit:

- Selected model profile: the profile currently selected in the UI.
- Running model profile: the profile snapshot used to start the current managed process.
- Restart required: the managed process is running, and the selected runtime profile differs from the running profile.

Runtime-affecting fields are:

- `modelID`
- `host`
- `serverPort`

Metadata fields such as `displayName`, `enableThinking`, and `notes` do not by themselves imply that the active server has changed.

## Switching Policy

Final v0.7 policy:

- While stopped, selecting a profile updates the active selection immediately.
- While stopped, Start launches the selected profile.
- While running, selecting another profile is allowed.
- While running, selection does not automatically stop or start `mlx_lm.server`.
- While running, if the selected runtime profile differs from the running profile, the UI shows `Restart required`.
- Restart applies the selected profile by using the existing Stop -> port release wait -> Start flow.
- Stop stops only the managed process, regardless of which profile is currently selected.

This policy keeps model switching explicit and avoids surprising server restarts while still letting users prepare the next model before restarting.

## UI Direction

- Highlight the selected profile in the model list.
- Show a compact `Running model` field when a managed server is active.
- Show `Restart required` when the selected profile and running profile differ.
- Keep detailed editing in the existing profile editor.
- Keep Connection Settings tied to the selected profile.
- Add explanatory log lines when:
  - selection changes while stopped,
  - selection changes while running,
  - Restart is required,
  - Restart applies the selected profile.

## Non-Goals

v0.7 does not add:

- Multiple simultaneous server management.
- Multiple model simultaneous startup.
- Proxy mode.
- Chat UI.
- LAN Web UI.
- App Intents.
- Auto unload.
- Hugging Face download manager.
- Model download.
- Model file deletion.
- Automated model existence checks.
- RAG.
- Embedding management.
- Tool-call translation.
- CI/CD.
- Notarization.
- DMG creation.
- App Store distribution.

## Safety Constraints

- Keep Direct Mode.
- Do not add a proxy.
- Do not add Chat UI.
- Do not send `/v1/chat/completions` from the app.
- Do not run model inference as part of model switching.
- Do not start `mlx_lm.server` merely because a model was selected.
- Restart must use only the existing managed process flow.
- Stop must target only the process held by this app.
- Do not stop external processes.
- Do not use `pkill`, `killall`, or `pgrep`.
- Do not delete model files, Hugging Face cache, or local model directories.
- Keep `settings.json` and `models.json` outside Git.
- Keep model files outside Git.
- Keep `.app` bundles and build artifacts outside Git.
- Do not document personal fixed paths.
- Use placeholders such as `<model-id>` and `<path-to-model>` when examples need paths or identifiers.
