# v0.1 Release Verification Checklist

Use this checklist before publishing or distributing MLX Server Manager v0.1.

The v0.1 release is Direct Mode only:

```text
OpenAI-compatible client -> mlx_lm.server
```

The app must not add a proxy, chat UI, alternate inference backend, or any app-side inference request.

## Environment

- [ ] Use a local macOS machine with `mlx_lm.server` available.
- [ ] Prefer `127.0.0.1` for v0.1 local use.
- [ ] Do not expose the server port directly to the internet.
- [ ] Confirm the model needed for manual verification is already available to `mlx_lm.server`.
- [ ] Confirm no secret token is needed by the app at runtime.

## First Launch

- [ ] Open the app from Xcode or the built app bundle.
- [ ] Confirm the main window opens without crashes.
- [ ] Confirm the model list is visible.
- [ ] Confirm the default model includes `unsloth/Qwen3.6-35B-A3B-UD-MLX-4bit`.
- [ ] Confirm status, connection settings, action buttons, and logs are visible.

## Settings

- [ ] Enter the `mlx_lm.server executable path` in the settings area.
- [ ] Confirm the path is not a hardcoded user-specific absolute path in source or docs.
- [ ] Confirm default host is `127.0.0.1`.
- [ ] Confirm default port is `8080`.
- [ ] Click `Save Settings`.
- [ ] Confirm logs show a successful settings save.
- [ ] Quit and relaunch the app.
- [ ] Confirm `settings.json` and `models.json` are restored by the app.
- [ ] Confirm the executable path and model settings are still visible after relaunch.

## Start

- [ ] Ensure the selected host and port are free before starting.
- [ ] Click `Start`.
- [ ] Confirm logs show the command summary without secrets.
- [ ] Confirm logs show a managed process pid.
- [ ] Confirm status progresses through starting/loading and reaches ready when the server is responsive.
- [ ] Confirm the app starts only `mlx_lm.server` directly and does not launch LM Studio, Ollama, llama.cpp, or a proxy.

## Ready Check

- [ ] Click `Check Ready` while the server is running.
- [ ] Confirm the app checks `GET /v1/models`.
- [ ] Confirm HTTP 200 from `/v1/models` marks the model as ready.
- [ ] Confirm the app does not send `/v1/chat/completions` as part of readiness or background checks.
- [ ] Stop the server, then click `Check Ready`.
- [ ] Confirm logs show a connection failure or non-ready result without crashing.

## Memory Display

- [ ] Start the server from the app.
- [ ] Confirm status shows the managed pid.
- [ ] Confirm memory usage appears as `Memory: xx.xx GB`.
- [ ] Wait at least one refresh interval.
- [ ] Confirm memory usage updates.
- [ ] Confirm memory monitoring targets only the pid started by this app.

## Restart

- [ ] Start the server and confirm it reaches ready.
- [ ] Note the current managed pid.
- [ ] Click `Restart`.
- [ ] Confirm logs show restart requested.
- [ ] Confirm the existing managed process is stopped first.
- [ ] Confirm logs show `port released`.
- [ ] Confirm a new process starts after the port is released.
- [ ] Confirm the new pid is different from the previous pid.
- [ ] Confirm ready check succeeds through `/v1/models`.

## Stop

- [ ] Click `Stop`.
- [ ] Confirm logs show stop requested for the managed pid.
- [ ] Confirm the process exits.
- [ ] Confirm logs show `port released`.
- [ ] Confirm status becomes stopped.
- [ ] Confirm pid display clears or shows no attached process.
- [ ] Confirm memory display becomes `Not running`.
- [ ] Confirm the app does not stop externally launched `mlx_lm.server` processes.

## Port Check

- [ ] With no server running, click `Check Port`.
- [ ] Confirm logs show the selected host/port is available.
- [ ] Start the managed server.
- [ ] Click `Check Port`.
- [ ] Confirm logs show the selected host/port is busy.
- [ ] Stop the managed server.
- [ ] Click `Check Port`.
- [ ] Confirm logs show the selected host/port is available again.

## Logs

- [ ] Confirm Start, Ready, Restart, Stop, Port, Memory, Settings, and Process messages appear in logs.
- [ ] Confirm logs remain readable after several actions.
- [ ] Click `Clear Logs`.
- [ ] Confirm the log view clears.
- [ ] Confirm `[info] logs cleared` appears after clearing.
- [ ] Confirm later actions append new logs normally.

## OpenAI-Compatible Copy Actions

- [ ] Click `Copy Base URL`.
- [ ] Paste into a text editor and confirm it is `http://127.0.0.1:8080/v1` for the default local configuration.
- [ ] Click `Copy Model ID`.
- [ ] Confirm the copied value matches the selected model ID.
- [ ] Click `Copy OpenAI-compatible config`.
- [ ] Confirm the copied JSON includes `base_url`, `api_key`, and `model`.
- [ ] Confirm the API key is a placeholder such as `not-required-local`.
- [ ] Click `Copy curl /v1/models`.
- [ ] Confirm the copied command targets `/v1/models`.
- [ ] Click `Copy curl /v1/chat/completions`.
- [ ] Confirm the copied command is only an example for external clients.
- [ ] Confirm the copied chat example includes `chat_template_kwargs.enable_thinking=false` for Qwen thinking control.
- [ ] Confirm the app itself does not execute the copied chat completion request.

## Repository Hygiene

- [ ] Run `git status --short`.
- [ ] Confirm `settings.json` is not tracked.
- [ ] Confirm `models.json` is not tracked.
- [ ] Confirm model files are not tracked.
- [ ] Confirm `.venv`, `models`, `logs`, `.env`, and `HF_TOKEN` are not tracked.
- [ ] Confirm model file extensions such as `*.safetensors`, `*.gguf`, and `*.bin` are ignored.
- [ ] Confirm README, docs, and Swift source do not contain user-specific absolute paths.
- [ ] Confirm Swift source does not hardcode local home directory paths.

## Scope Guardrails

- [ ] Confirm Direct Mode is maintained.
- [ ] Confirm there is no proxy in the inference path.
- [ ] Confirm there is no chat UI.
- [ ] Confirm the app does not implement LAN Web UI.
- [ ] Confirm the app does not implement App Intents.
- [ ] Confirm the app does not implement auto unload.
- [ ] Confirm the app does not implement a Hugging Face download manager.
- [ ] Confirm the app does not manage multiple simultaneous servers in v0.1.
- [ ] Confirm Ready checks use `/v1/models` only.
- [ ] Confirm `/v1/chat/completions` appears only as copied client example text, not as an app-side request.

## Final Manual Pass

- [ ] Start the app.
- [ ] Check Port.
- [ ] Start the managed server.
- [ ] Confirm Ready.
- [ ] Confirm Memory.
- [ ] Restart and confirm a new pid.
- [ ] Stop and confirm port release.
- [ ] Quit and relaunch.
- [ ] Confirm settings restore.
- [ ] Confirm no unexpected Git changes were created.
