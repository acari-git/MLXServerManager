# v0.3 Plan

v0.3 focuses on Model profile editing for the existing Direct Mode server manager.

The goal is to let users edit the selected model profile from the UI and persist those edits to `models.json`, without changing the inference route or adding a proxy.

## Scope

### Included

- Edit the selected model profile in the UI.
- Edit `modelID`.
- Edit `displayName`.
- Edit `host`.
- Edit `serverPort`.
- Toggle `enableThinking`.
- Edit `notes`.
- Save edited model settings to `models.json`.
- Cancel unsaved edits.
- Validate required fields before saving.
- Refresh Connection Settings, Copy Config, and curl output after saving.
- Use edited `modelID`, `host`, and `serverPort` for Start.
- Guard edits while a managed server process is running.

### Deferred

- Adding or deleting multiple model profiles.
- Multiple simultaneous server management.
- Hugging Face download manager.
- Model file deletion.
- Proxy mode.
- Chat UI.
- LAN Web UI.
- App Intents.
- Auto unload.
- Running `/v1/chat/completions` from the app.

## Direct Mode

v0.3 must preserve Direct Mode:

```text
OpenAI-compatible client -> mlx_lm.server
```

Editing model settings changes only local configuration and future managed process launches. It must not insert the app into the inference path.

## Running Process Policy

Editing `modelID`, `host`, or `serverPort` while a managed process is running can create a mismatch between the UI and the running server.

Two implementation options are acceptable:

1. Disable important fields while a managed process is running.
2. Allow saving but show that Restart is required before the running server uses the new values.

The safer v0.3 default should be option 1: disable `modelID`, `host`, and `serverPort` while a managed process is running. This avoids implying that the running process has changed when only `models.json` has changed.

Fields that do not affect the running process, such as `displayName`, `enableThinking`, and `notes`, may remain editable if the UI clearly separates runtime-affecting fields from metadata fields.

## Validation

Before saving a profile:

- `modelID` must not be empty.
- `host` must not be empty.
- `serverPort` must be between 1 and 65535.
- Whitespace-only values should be treated as empty.

Invalid edits should not be written to `models.json`. The UI should show a concise validation message and write a clear Logs entry.

## Persistence

Saving model profile edits should use the existing settings persistence path:

- `models.json`

The file is local runtime state and must not be committed.

## Safety Rules

- Keep Direct Mode.
- Do not add a proxy.
- Do not add Chat UI.
- Do not run `/v1/chat/completions` from the app.
- Do not launch `mlx_lm.server` from profile editing.
- Do not stop external processes from profile editing.
- Do not use `pkill`, `killall`, or `pgrep`.
- Do not hardcode user-specific absolute paths.
- Do not commit `settings.json`, `models.json`, model files, logs, `.env`, `HF_TOKEN`, or virtual environments.

## Expected Outcome

A user should be able to edit the selected model profile, save valid changes, and immediately see Connection Settings and copied config reflect the new values. Start should use the saved model ID, host, and port on the next launch.
