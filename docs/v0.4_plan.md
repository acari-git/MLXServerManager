# v0.4 Plan

v0.4 focuses on Menu bar quick actions for the existing Direct Mode server manager.

The goal is to let users quickly see managed server status and run common management actions from the macOS menu bar without changing the inference route or adding background inference behavior.

## Scope

### Included

- Add a macOS menu bar status item.
- Show the current managed server status:
  - `stopped`
  - `starting`
  - `ready`
  - `stopping`
  - `failed`
- Start the managed server from the menu bar.
- Stop the managed server from the menu bar.
- Restart the managed server from the menu bar.
- Run Setup Diagnostics from the menu bar.
- Open or focus the main app window from the menu bar.
- Open or expose Connection Settings from the menu bar.
- Quit the app from the menu bar.
- Reuse existing `AppViewModel`, process management, diagnostics, and connection settings behavior.
- Preserve Direct Mode for all menu bar actions.

### Deferred

- Adding or deleting model profiles.
- Multiple simultaneous server management.
- Proxy mode.
- Chat UI.
- LAN Web UI.
- App Intents.
- Auto unload.
- Hugging Face download manager.
- Model download.
- Model file deletion.
- Running inference from the app.
- Full log viewer redesign.
- Distribution build and notarization.

## Direct Mode

v0.4 must preserve Direct Mode:

```text
OpenAI-compatible client -> mlx_lm.server
```

Menu bar actions are shortcuts to existing management actions. They must not insert the app into the inference path.

## UI Direction

The menu bar surface should stay lightweight:

- Status can be represented with concise text or a simple icon.
- Start, Stop, Restart, and Run Diagnostics should be direct actions.
- Open App should focus the main window.
- Connection Settings may either open the main window near the existing connection panel or show a compact summary.
- Detailed model editing, settings editing, and full log review should remain in the main window.
- v0.4 does not need polished custom icons, launch-at-login behavior, or log viewer expansion.

## Safety Rules

- Keep Direct Mode.
- Do not add a proxy.
- Do not add Chat UI.
- Do not run `/v1/chat/completions` from the app.
- Do not run model inference from menu bar actions.
- Do not start `mlx_lm.server` only to update menu bar status.
- Stop only the managed process held by this app.
- Do not stop externally launched `mlx_lm.server` processes.
- Do not use `pkill`, `killall`, or `pgrep`.
- Do not add multiple simultaneous server management.
- Do not add a Hugging Face download manager.
- Do not add Auto unload.
- Do not add LAN Web UI.
- Do not add App Intents.
- Do not hardcode user-specific absolute paths.
- Do not commit `settings.json`, `models.json`, model files, logs, `.env`, `HF_TOKEN`, or virtual environments.

## Expected Outcome

A user should be able to glance at the macOS menu bar for managed server status, run Start, Stop, Restart, and Run Diagnostics, open the main window, view or reach connection settings, and quit the app. All behavior should reuse the existing management path and maintain Direct Mode.
