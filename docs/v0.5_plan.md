# v0.5 Plan

v0.5 focuses on documenting distribution build steps for personal local use.

The goal is to make it clear how to build MLX Server Manager safely on a user's own Mac and run it as a normal macOS app, without changing the app's Direct Mode behavior or adding automated packaging.

## Scope

### Included

- Explain Debug build and Release build differences.
- Document manual build steps from Xcode.
- Document CLI build steps with `xcodebuild`.
- Document where the `.app` bundle is produced.
- Describe local personal-use distribution assumptions.
- Explain Gatekeeper, signing, and notarization caveats.
- Explain Apple Developer Program considerations for personal use.
- Explain why App Sandbox is disabled.
- Document prerequisites for launching and stopping `mlx_lm.server`.
- Clarify that the `mlx_lm.server` executable path must be configured in the app UI.
- Document where `settings.json` and `models.json` are stored.
- Clarify that model files are not bundled with the app.
- Define whether a GitHub Release should include a `.app` bundle.
- Keep automatic packaging deferred.

### Deferred

- Performing notarization.
- Apple Developer Program based formal distribution.
- DMG creation.
- Sparkle or other automatic updates.
- Homebrew cask.
- App Store distribution.
- CI/CD.
- GitHub Actions.
- Bundling model files.
- Automatic `mlx-lm` installation.
- Hugging Face download manager.
- Proxy mode.
- Chat UI.
- LAN Web UI.
- App Intents.
- Auto unload.

## Direct Mode

v0.5 must preserve Direct Mode:

```text
OpenAI-compatible client -> mlx_lm.server
```

Distribution build documentation must not introduce a proxy, chat flow, or app-side inference request.

## Distribution Position

v0.5 should prioritize a clear personal-build workflow:

1. Build locally on the user's Mac.
2. Configure `mlx_lm.server executable path` in the app UI.
3. Use the app to manage only the process it starts.
4. Keep models, runtime settings, logs, and build artifacts outside Git.

GitHub Release attachment policy should be conservative. If a `.app` bundle is attached later, the release notes must explain signing and Gatekeeper limitations. v0.5 should document the policy, not automate it.

## App Sandbox Rationale

App Sandbox is disabled because the app needs to launch and stop a local `mlx_lm.server` executable selected by the user and capture basic process output for Logs.

This does not mean the app should broaden behavior beyond local process management. Stop must still target only the managed process held by the app. The app must not stop external `mlx_lm.server` processes.

## Safety Rules

- Keep Direct Mode.
- Do not add a proxy.
- Do not add Chat UI.
- Do not run `/v1/chat/completions` from the app.
- Do not require model inference as part of distribution verification.
- Do not include model files in the repository or release assets.
- Do not commit `settings.json` or `models.json`.
- Do not commit `.app` bundles or build artifacts.
- Do not hardcode user-specific absolute paths.
- Use placeholders such as `<path-to-mlx_lm.server>` in examples.
- Keep App Sandbox rationale narrow and tied to local managed process control.
- Stop only the managed process held by this app.
- Do not stop external processes.
- Do not use `pkill`, `killall`, or `pgrep`.

## Expected Outcome

A user should be able to follow `docs/distribution_build.md` to create a local Debug or Release build, find the resulting `.app`, understand signing and Gatekeeper limitations, configure `mlx_lm.server`, and run the app locally without committing runtime files, model files, or build artifacts.

## Verified Release Build Notes

v0.5 Release build verification has confirmed that the documented `xcodebuild` flow works with:

- Scheme: `MLXServerManager`
- Configuration: `Release`
- Derived data path: `/tmp/MLXServerManagerReleaseDerivedData`
- Signing override: `CODE_SIGNING_ALLOWED=NO`

The build completed with `BUILD SUCCEEDED`. The observed app bundle path was:

```text
/tmp/MLXServerManagerReleaseDerivedData/Build/Products/Release/MLXServerManager.app
```

The observed app bundle size was `916K`.

The app launched with `open -n`, the `MLXServerManager` process was visible through `System Events`, and the app quit normally. The final Git status was clean, and no `.app`, build artifact, runtime settings file, or model file was tracked by Git.

This remains an unsigned local personal-use verification. Notarization, DMG creation, GitHub Release asset creation, and CI/CD remain deferred.
