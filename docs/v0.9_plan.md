# v0.9 Plan

## Focus

v0.9 documents how to package an unsigned Release `.app` as a `.zip` file for a GitHub Release asset.

This is a local-use distribution plan. It does not add code, change Xcode project settings, create build artifacts, create a zip file, or create a tag during planning.

Direct Mode remains unchanged:

```text
OpenAI-compatible client -> mlx_lm.server
```

The app continues to manage `mlx_lm.server` directly. Packaging documentation must not introduce a proxy, Chat UI, telemetry, model inference, or runtime automation.

## Goals

- Document how to create a Release build.
- Document how to zip the unsigned `.app`.
- Document that the zip should contain only `MLXServerManager.app`.
- Document GitHub Release asset attachment policy.
- Clarify that the build is unsigned and intended for local use.
- Explain Gatekeeper, quarantine, signing, and notarization caveats.
- Keep `.app`, `.zip`, `.dSYM`, derived data, and build artifacts out of Git.
- Keep runtime settings, model profiles, model files, and Hugging Face cache out of the zip.
- Provide a manual checklist for packaging and release asset verification.

## Expected Workflow

1. Create a Release build.
2. Confirm the `.app` exists.
3. Create a `.zip` using `ditto -c -k --keepParent`.
4. Confirm the zip contains only the `.app` bundle.
5. Confirm the zip size.
6. Confirm `.app`, `.zip`, `.dSYM`, derived data, runtime settings, and model files are not tracked by Git.
7. Attach the zip to a GitHub Release as a Release asset.
8. Include user-facing release notes that explain unsigned local-use status and Gatekeeper caveats.

## Non-Goals

- Notarization.
- Developer ID signing.
- DMG creation.
- Sparkle or other automatic updates.
- CI/CD.
- GitHub Actions.
- App Store distribution.
- Homebrew cask.
- Installer creation.
- Bundling runtime settings.
- Bundling model files.
- Bundling Hugging Face cache.
- Proxy mode.
- Chat UI.
- LAN Web UI.
- App Intents.
- Auto unload.
- Hugging Face download manager.
- Model download.
- Model file deletion.
- Multiple simultaneous server management.

## Safety Constraints

- Keep Direct Mode.
- Do not send `/v1/chat/completions` from the app.
- Packaging verification must not run model inference.
- Packaging verification must not start `mlx_lm.server`.
- Stop remains limited to the managed process held by the app.
- External processes must not be stopped.
- `pkill`, `killall`, and `pgrep` must not be used.
- `.app`, `.zip`, `.dSYM`, derived data, and build artifacts must stay outside Git.
- `settings.json` and `models.json` must stay outside Git and outside Release assets.
- Model files must stay outside Git and outside Release assets.
- `.env` and `HF_TOKEN` must stay outside Git and outside Release assets.
- Documentation must not include personal fixed paths.

## Expected Outcome

At the end of v0.9, the repository should clearly document how to create an unsigned local-use zip artifact, how to verify its contents, and how to attach it to a GitHub Release without implying notarized or formally signed distribution.
