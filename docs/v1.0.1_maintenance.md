# v1.0.1 Maintenance Plan

v1.0.1 is a short maintenance phase after the v1.0.0 Stable Release. It is intended for light verification and small documentation or wording fixes before broader public announcement.

This phase must not expand product scope.

## Goals

- Re-download and verify the published v1.0.0 GitHub Release asset.
- Confirm the unsigned zip launches after extraction.
- Review first-run documentation for unclear steps.
- Review Gatekeeper warning guidance.
- Document the observed macOS "damaged and can't be opened" warning for the unsigned v1.0.0 asset.
- Review setup guidance when `mlx_lm.server executable path` is not configured.
- Review model profile setup guidance for missing or incorrect profile values.
- Run a lightweight regression pass for Start, Stop, Restart, Diagnostics, Logs, Menu bar, and Connection Settings.
- Keep room for README and docs wording fixes.

## Allowed Changes

- README clarifications.
- Docs clarifications.
- Known limitations additions.
- Release asset verification procedure clarifications.
- First-run workflow clarifications.
- Gatekeeper warning clarifications.
- Small UI wording fixes.
- Small error-message wording fixes.

## Release Asset Re-Download Verification

After publishing v1.0.0, verify the GitHub Release asset as a user would:

1. Download the release asset from GitHub in a browser.
2. Confirm the asset name is `MLXServerManager-v1.0.0-unsigned.zip`.
3. Confirm the zip contains only `MLXServerManager.app/`.
4. Confirm the zip does not contain:
   - `settings.json`
   - `models.json`
   - model files
   - model directories
   - `.env`
   - `HF_TOKEN`
   - `.dSYM`
   - DerivedData
   - logs
   - Hugging Face cache
   - AppleDouble `._*` metadata files
5. Extract the zip into a temporary folder.
6. Launch the extracted app with `open -n`.
7. If macOS says the app is damaged and must be moved to the Trash, treat it as a possible Gatekeeper quarantine warning for the unsigned, non-notarized app.
8. Verify the asset name, zip contents, and checksum before removing quarantine.
9. If appropriate for local verification, remove quarantine from the extracted app:

   ```sh
   xattr -dr com.apple.quarantine /path/to/MLXServerManager.app
   open -n /path/to/MLXServerManager.app
   ```

10. Quit the verification app process.
11. Confirm no verification process remains running.

## First-Run Review

Confirm the README and docs make these steps clear:

- Users must provide their own `mlx-lm` environment.
- Users must configure `mlx_lm.server executable path` in Settings.
- Users must configure or add a Model Profile.
- Users should run Setup Diagnostics before Start.
- Users should use Connection Settings to copy Base URL, Model ID, JSON config, and curl examples.
- OpenAI-compatible clients connect directly to `mlx_lm.server`.
- Direct Mode means the app is not in the inference request path.

## Lightweight Regression Pass

The v1.0.1 maintenance pass should lightly confirm:

- Start.
- Stop.
- Restart.
- Port Check.
- Ready Check via `/v1/models`.
- Run Diagnostics.
- Diagnostics summary.
- Logs readability.
- Copy Logs.
- Clear Logs.
- Menu bar Start, Stop, Restart, Run Diagnostics, Open App, and Quit.
- Connection Settings copy actions.

## Safety Boundaries

- Direct Mode remains unchanged.
- The app is not in the inference request path.
- The app must not execute `/v1/chat/completions`.
- Stop and Restart target only the app-managed process.
- Swift code must not use `pkill`, `killall`, or `pgrep`.
- Model files, runtime settings, and secrets must not be committed.
- `.app`, `.zip`, `.dSYM`, and build artifacts must not be committed.
- Documentation must not include personal fixed paths.

## Exit Criteria

- v1.0.0 release asset has been re-downloaded and verified.
- First-run documentation has been reviewed.
- Gatekeeper warning text, including the "damaged and can't be opened" quarantine case, is understandable.
- Known limitations remain visible.
- No new scope has entered v1.0.1.
- Any changes are limited to allowed maintenance fixes.
